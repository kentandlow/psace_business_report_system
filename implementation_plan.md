# 実装計画 (Implementation Plan)

## 1. システム要件
- **実行タイミング**: 毎週月曜日 朝9:00 (JST)
- **対象データ**: 直近1週間（先週）にリリースされたあらゆる宇宙関連の情報ソース
- **出力形式**: スライド資料（目安として全体で30枚程度のボリューム感を満たすこと）
- **出力先**: 指定のWebサイト（HP）への自動掲載（HTML形式などデプロイしやすい形を推奨）
- **AI連携**: 情報の収集・要約・考察に Gemini API を活用する

## 2. スライドの構成要件
以下の構成で構成された30枚のスライドレポートを作成すること：
1. **宇宙政策ニュース**: 日・米・欧・中の宇宙政策に関する直近ニュース一覧、サマリー、出典
2. **最新研究内容**: 宇宙関連の最新研究内容、出典
3. **ビジネスニュース**: 日・米・欧・中の宇宙関連ビジネスに関する最新ニュース一覧、サマリー、出典
4. **資金調達ニュース**: 日・米・欧・中のスタートアップ資金調達ニュース一覧、サマリー、出典
5. **考察と展望**: 上記の情報を踏まえた考察、今後のビジネス展望、投資家として注目すべきビジネス領域

## 3. 推奨技術スタック
Claude Codeが実装・自動実行しやすい構成を構築してください。
- **言語**: Python または Node.js (TypeScript)
- **定期実行処理**: GitHub Actions（`cron: '0 0 * * 1'` などを指定）または AWS EventBridge 等によるスケジュール実行を推奨
- **情報収集・分析**:
  - NewsAPI, Google Custom Search APIによるウェブ検索、もしくは RSSフィード自動巡回
  - `Google Gemini API` を用いた高度な要約とインサイト生成・スライド構成案策定
- **スライド生成**:
  - Node.js であれば `Marp` や `Slidev`、`reveal.js` を用いて Markdown からスライド形式のHTMLを生成
  - Python であれば `python-pptx` を用いた PPTX出力
- **HP掲載**: GitHub Pages、Vercel などへの自動デプロイ設定を組み込む

## 4. 承認フローの実装
- システムの根幹ロジック（たとえば使用するAPIの変更、スライド生成エンジンの変更、主要モジュールの削除など）をClaude Codeが変更・実行する際は、スクリプトやプロンプトによって動作を中断し、「変更内容」と「影響範囲」を**平易な言葉で出力**した上で、ユーザーの[Y/N]入力を待つ設計を徹底すること。
