# タスク一覧 (Tasks)

Claude Codeは以下のステップに従ってシステムを構築してください。
**必ず先に `plan/` ディレクトリ配下にある各ステップの詳細実装計画書（`01_overview.md` ～ `05_deployment.md`）を読んでから実装を開始すること。**

- [x] **Step 1: プロジェクトの初期化とセキュリティ保全**
  - プロジェクトディレクトリの初期化
  - `.gitignore` の作成（`.env` を必ず除外）
  - Gemini APIキー管理用の `.env.example` ファイル作成、および環境変数読み込み設定の実装

- [ ] **Step 2: 情報収集モジュールの実装**
  - 前週1週間分のデータを取得するプログラムの実装
  - 対象: 日米欧中の宇宙政策、最新研究、ビジネスニュース、資金調達ニュース
  - 収集先URL（出典）も必ず保持する仕組みを作ること

- [ ] **Step 3: Gemini API による分析・要約モジュールの実装**
  - 収集したテキストデータを Gemini API に渡し、指定されたスライド構成（全30枚相当）に合わせて要約と全体考察・投資家向け展望を生成するプロンプトと処理を実装

- [ ] **Step 4: スライド生成モジュールの実装**
  - 分析モジュールから受け取った構造化データ（MarkdownやJSON等）から、プレゼンテーション用スライドデータ（HTMLまたはPPTX）を生成する処理の実装

- [ ] **Step 5: Webサイト（HP）公開モジュールの実装**
  - 生成されたスライドをHP（GitHub Pages等のホスティング環境）に自動で配置・デプロイするフローの構築

- [ ] **Step 6: スケジューラ（定期実行）の設定**
  - 毎週月曜日の朝9:00（JST）に自動で一連の処理が走るように、GitHub Actions 等のワークフローファイル（`.github/workflows/...`）を構築

- [ ] **Step 7: ユーザー承認プロセスの確立**
  - システムの中核ファイル修正や削除を行う際、ユーザーに何が変わるか分かりやすく説明し承認を求める仕組みが遵守されているか、動作確認テストを行う

**🚨 Claude Codeへの警告 🚨**
作業を進行する際、仕様外のツール追加や既存コードの大きな削除など「システムの根幹」に影響を与える行動を取る前には、**必ずユーザーに承認を求めてください**。その際、影響を技術者以外にもわかる平易な言葉で説明すること。
